Documentação da API

Autenticação

Para autenticar um usuário, você precisa passar pelo header Authorization, uma chave de autenticação nesse formato: "Basic login:senha". O login e senha precisam estar criptografados em Base64 (utilizando a função btoa do JavaScript).

Exemplo: Authorization: 'Basic ' + btoa('login:senha')

https://{Request.host}/user

Método: POST

Autenticação necessária: Não

Descrição: Cria um novo usuário no sistema.

Corpo da Requisição:

login: string - Nome de usuário desejado.

email: string - Endereço de email do usuário.

senha: string - Senha do usuário.

Resposta: Retorna um código HTTP 200 em caso de sucesso.

https://{Request.host}/user/current-user

Método: GET

Autenticação necessária: Sim

Descrição: Retorna dados do usuário autenticado.

Corpo da Requisição:

login: string - Nome de usuário do usuário existente.

senha: string - Senha do usuário.

Resposta: Retorna o status de autenticação do usuário em formato JSON.

https://{Request.host}/user/count

Método: PUT

Autenticação necessária: Sim

Descrição: Contabiliza os pontos das alternativas passadas.

Corpo da Requisição:

alternativas: Array(integer) - IDs das questões que o usuário respondeu.

Resposta: Retorna o usuário autenticado com os dados atualizados.

https://{Request.host}/quiz/:id_disciplina

Método: GET

Autenticação necessária: Não

Descrição: Exibe todas as questões e alternativas de uma disciplina.

Parâmetros da Requisição:

id_disciplina: ID da disciplina

Resposta: Retorna todas as questões e alternativas de uma disciplina.

https://{Request.host}/quiz/disciplinas

Método: GET

Autenticação necessária: Não

Descrição: Exibe todas as disciplinas do curso de DS.

Resposta: Retorna todas as disciplinas do curso de DS.

https://{Request.host}/quiz/disciplina/:id_disciplina

Método: GET

Autenticação necessária: Não

Descrição: Exibe uma disciplina.

Parâmetros da Requisição:

id_disciplina: ID da disciplina

Resposta: Retorna as informações de uma disciplina.
